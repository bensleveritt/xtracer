About

maintained by Papadopoulos Nikos [nikpapas@gmail.com]

XTracer is a global illumination renderer written in c and c++.

Version 0.7 consisted of a basic raytracer with rundimentary functionality along with a simple parser for scene scripts and served as my thesis project for the B.Sc. of computer science at the University of Piraeus. Version 1.1 was developed as part of my dissertation for the MSc of computer graphics programming at the University of Hull.

Features

Implemented:


Primitives    : Plane, Triangle, Sphere, Mesh
Materials     : Lambert, Phong, Blinn-Phong
Lights        : Point, Sphere, Box, Triangle
Cameras       : Pinhole, Thin lense
Shading       : Reflection, Refraction, DoF, Soft shadows, Texture mapping
Acceleration  : Threads, Octrees, KD-trees
Anti-aliasing : Supersampling
Rendering     : HDR, Whitted/Distributed raytracing, Photon mapping (diffuse term)

Future expansion:


Shading       : Fresnel, Beer's law
Anti-aliasing : Adaptive SS, Jittering

Compilation / Installation

The source code is distributed along with a makefile configuration script for unix based systems with a GNU toolchain installed. To view all the available configuration options type: ./configure --help. Once the configuration is done, you can build the binary by typing: make. To install the binary in your system type: make install. To uninstall type: make uninstall. For windows users, a visual studio 2010 solution is available in the ide directory.

Dependencies

Supported platforms

xtracer should be able to compile and run on any system equipped with openmp compatible c and c++ compilers. It has been tested without any problems under the following platforms:

  • Linux
  • FreeBSD
  • MacOSX
  • MS Windows XP, 7
